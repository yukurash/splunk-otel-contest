Claude Code ネイティブ /cost ・ /usage の実出力（PII 伏字化済み）
----------------------------------------------------------------
目的: Splunk(OTel) と「何が出て何が出ないか」を実物で比較する。
取得: Claude Code TUI で /cost / /usage を実行（2026-06, Asia/Tokyo）。
重要な観測:
  - /cost と /usage は同一画面に統合されている（タブ: Settings/Status/Config/Usage/Stats）。
  - 日付トグル: 公式docは "d/w で過去24h/過去7日を切替可" と明記。だが実機(当時のバージョン)では
    効かず "Last 24h" 固定だった（=仕様上は切替可能だが、実機では効かなかった/バージョン依存）。
  - skills/subagents は出るが「% of usage」のみ。実額($)・実トークンは出ない。
  - 画面自身が "these are independent characteristics of your usage, not a breakdown"
    （＝内訳ではない）と明記し、"Approximate, based on local sessions on this machine" と断る。
================================================================

================================================================
所見:
- /cost で出た粒度        : セッション合計 + "Usage by model"（モデル別の $ とトークン）まで。
- /usage で subagent は出たか: 出た。ただし "% of usage" のみ（$/トークン無し）。さらに
                              Subagents 欄に skill 名(commit-convention)が混入し近似的。
                              frontend-dev は 1% = 直近24hの大半がこの分析セッションに食われ、
                              A/Bビルドの subagent コストとして isolate できていない。
- /usage で skill は出たか  : 出た。/quality-gate 2% / /commit-convention 1%（% のみ）。
- 実額($)が出るか / %のみか : skill/subagent は %のみ。$ はモデル別とセッション合計のみ。
- A/B(run)で分けられるか    : 不可。run=quality / run=cost という自分で付けた次元の概念が無い。
- 期間を選べるか            : 公式docは d/w で 24h/7d 切替可だが、実機では Last 24h 固定だった。
- このマシン限定か          : はい。"local sessions on this machine, does not include other devices"。
- トレース(waterfall)は見えるか: 画面では見えない（委譲スパン/ネストの概念なし）。※OTelトレース層では可。
- 画面の自己申告            : "Approximate" かつ "not a breakdown"（＝正確な内訳ではないと明言）。

結論: ネイティブ /cost・/usage では「特定の Agent / Skill の "実コスト・実トークン" を、
      測定対象だけ isolate して、A/B で並べて」見ることはできない。出るのは
      直近24h・このマシン・近似%の "not a breakdown"。
      実コスト・実トークンを agent/skill 別に取るには OTel の3層を使う:
        1) メトリクス (agent.name × skill.name × query_source × model) = 集計・ダッシュボード向き
        2) イベント   (api_request の cost_usd + 属性 + prompt.id)      = リクエスト単位の明細・監査向き
        3) トレースbeta(agent_id / query_source / per-span tokens)     = 同一モデルの custom subagent 分離 + waterfall
      本企画は 1)メトリクス・2)イベント・3)トレース の3層すべて取得済み
      （2)イベントのサンプル: captures/api-request-events-redacted.txt）。

前提: 直接 Anthropic API 認証時のみ claude_code.cost.usage が機能する。
      Bedrock/Vertex/Foundry では Claude Code はコストメトリクスを送らない（公式。LiteLLM 等で代替）。
