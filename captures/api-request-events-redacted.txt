イベント層: claude_code.api_request の実捕捉サンプル（PII伏字化済み）
----------------------------------------------------------------
取得方法:
  - .claude/settings.json に OTEL_LOGS_EXPORTER=otlp を追加。
  - Collector に logs パイプライン（receivers[otlp] → redaction/scrub → debug exporter）を追加。
  - 小セッションを1回実行:
      claude -p "target-app の App.tsx と useHabits.ts を読んで役割を1行ずつ説明" --allowedTools Read
  - Collector debug 出力から claude_code.api_request の LogRecord を抽出し、prompt.id で集計。
注: user.email / user.id / organization.id / account_* は <redacted> 化（Collector の redaction は
    秘密パターン/ファイルパス向けで、email 等は素通りするため手動伏字化した）。
================================================================

########## 2) prompt.id で束ねた「1指示のファンアウト」##########
# 1つのユーザー指示が 3 本の API 呼び出しに展開し、合計 $0.106 かかった内訳

prompt.id = ce4b8a45-...-21fd3b3b3090   ( 3 api_request / 合計 $0.10600 )

  model                       query_source              cost_usd   in     out  cacheRead cacheCreate  dur
  claude-haiku-4-5            generate_session_title    $0.00067   550    24   0         0            1847ms
  claude-opus-4-8             sdk                       $0.04849   2996   175  11447     2341         4254ms
  claude-opus-4-8             sdk                       $0.05684   2       167  13788     4576         4788ms
  --------------------------------------------------------------------------------------------------------
  合計                                                  $0.10600

読み取れること:
  - 「2ファイル読んで説明して」という 1 指示が、内部で 3 リクエストに展開（=ファンアウト）。
    ・haiku 1本はセッションタイトル自動生成（query_source=generate_session_title）= 補助。
    ・opus 2本が実作業（query_source=sdk）。
  - メトリクス(cost.usage 合算)では「opus がいくら」までしか出ないが、イベントでは
    「どの request_id が $0.05684」という 1 リクエスト単位の実コスト明細が取れる。
  - prompt.id で「1つの指示が起こした総コスト」を束ねられる（$0.106/指示）。
  - イベントの query_source はメトリクス(main/subagent/auxiliary)より粒度が細かく、
    generate_session_title / sdk / compact / repl_main_thread 等のサブシステム名が出る。


########## 3) これで開く分析（メトリクス/トレースにできないこと）##########
  - per-request の実コスト明細・監査ログ（cost_usd を行単位で保持）。
  - prompt.id で「高コストな指示」をランキングできる。
  - request_id を鍵に トレースの claude_code.llm_request スパン（トークンは持つが cost_usd は無い）と
    join → 「各スパンの実コスト($)」= waterfall の各バーに金額を載せられる。
