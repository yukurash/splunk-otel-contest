イベント層: claude_code.api_request の実捕捉サンプル（run=zenn / PII伏字化済み）
----------------------------------------------------------------
取得方法:
  - .claude/settings.json に OTEL_LOGS_EXPORTER=otlp を追加。
  - Collector に logs パイプライン（receivers[otlp] → redaction/scrub → debug exporter）を追加。
  - 習慣トラッカーのビルド(tech-lead 1指示)を実行し、Collector debug 出力から
    claude_code.api_request の LogRecord を抽出、prompt.id で集計。
  - さらに scripts/events-bridge.mjs で ingest /v2/event に転送し、Splunk Event Feed に着弾させた
    （signalfx exporter は素の OTel ログをイベント化しないため、ブリッジで /v2/event に投げ直す必要がある）。
注: user.email / user.id / organization.id / account_* / session.id は <redacted> 化。
    （Collector の redaction は秘密パターン/ファイルパス向けで email 等は素通りするため手動伏字化）
================================================================

########## 2) prompt.id で束ねた「1指示のファンアウト」##########
# 「習慣トラッカーを作って」という tech-lead への1指示が、内部で 102 本の API 呼び出しに展開した

prompt.id = ea0eda6f-...   ( 102 api_request / 合計 $1.7575 )

  model              query_source             cost_usd    in     out    cacheRead
  claude-haiku-4-5   generate_session_title   $0.00115    1009   29     0          ← 補助(タイトル生成)
  claude-opus-4-8    sdk                       $0.08895   2789   203    0          ← 最高額の1本
  claude-opus-4-8    sdk                       $0.04531   2      106    6993
  claude-opus-4-8    sdk                       $0.01532   2      195    10908
  claude-opus-4-8    sdk                       $0.04948   2      953    11765
  ...（以下、opus / sonnet / haiku 合わせて 102 本）
  --------------------------------------------------------------------------------
  合計                                         $1.7575

読み取れること:
  - 「アプリ作って」という 1 指示が、内部で 102 リクエストに展開（=ファンアウト）。
  - メトリクス(cost.usage 合算)では「opus がいくら」までしか出ないが、イベントでは
    「どの request_id が $0.08895」という 1 リクエスト単位の実コスト明細が取れる。
  - prompt.id で「1つの指示が起こした総コスト」を束ねられる（$1.7575 / 指示）。
  - 各行の request_id を鍵に、トレースの claude_code.llm_request スパン（トークンは持つが
    cost_usd は無い）と join → waterfall の各バーに金額を載せられる。


########## 3) これで開く分析（メトリクス/トレースにできないこと）##########
  - per-request の実コスト明細・監査ログ（cost_usd を行単位で保持）。
  - prompt.id で「高コストな指示」をランキングできる。
  - request_id で トレースのスパンと join → 各スパンの実コスト($)を可視化。
